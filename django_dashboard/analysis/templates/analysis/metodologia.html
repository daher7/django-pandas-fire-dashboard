DOCUMENTACIÓN TÉCNICA

Arquitectura del Sistema

FireWatch AI opera bajo una infraestructura distribuida, separando el almacenamiento, el servicio de datos y la interfaz de usuario.

01

Extracción

Obtención de registros brutos desde el dataset histórico y procesamiento inicial mediante la librería Pandas para la normalización de variables.

02

Persistencia

Almacenamiento en base de datos relacional SQLite, estructurando la información en tablas normalizadas de incendios y provincias.

03

Capa de API

Construcción de una API REST independiente con FastAPI que consulta la base de datos y sirve los datos en formato JSON.

04

Consumo

El backend de Django actúa como cliente, consumiendo los endpoints de la API de FastAPI para procesar la lógica de negocio.

05

Visualización

Generación de componentes visuales mediante Plotly, renderizando mapas y gráficos dinámicos integrados en la interfaz de usuario.
